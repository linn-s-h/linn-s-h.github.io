Garden Paradise

In Spring 2024, I spent a semester in South Korea on exchange. Garden Paradise was a collaborative team project in my all-time favorite course — Relational Databases. It's a plant search tool for beginner plant owners looking to expand their collection and need help narrowing down "next plant" options. The intention is to let users expand their plant knowledge and dictionary, and save their searches for future reference. This project was developed using Python and SQL, with Tkinter for GUI. Our data comes from Queensland’s Government’s ‘Waterwise Plants’ dataset.

Main Functionalities

  • Quick search for plants based on user input (e.g., plant type, size, light requirements)
  • Advanced search, allowing users to play around with different results
  • View detailed information about each plant, including care instructions and images
  • Save favorite plants for future reference
  • Filter plants by specific criteria to help users find the best options for their needs
  • An user account attached to the user's saved favorites

Though this was a mainly coding project about databases, we made sure to also have design in mind.

Throughout the project we made small, yet intentional design choices:

  • Vintage aesthetic, mocking the interface of a shop system.
  • Password is hidden.
  • Handling possible user errors, mostly through invalid database interactions. Such as icomplete sign-up, existing username, unmatched passwords, and access to saved favorites.
  • Notes is seperated, to avoid information overload.

We had an iterative process and made some design changes

I conducted usability tests on friends and my roommates, in different iterations of the app. To discover new errors throughout the development phase, we continously changed participant.

Potential Improvements

Due to the project’s time constraints, we down-prioritized making bigger design changes, even though there were a few painstakingly obvious ones.

  • Difficult to differenciate between Quick Search and Advanced Search
  • Initially we also wanted a top bar for users to sort plants by maintenance, height ranges and spread ranges. This would be especially helpful for new plant owners, but we made a compromise to prioritize other main functions to fulfill the project’s formal requirements
